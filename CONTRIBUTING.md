# Contributing to Hardcoded String Finder

Thank you for your interest in contributing! 🎉

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your feature: `git checkout -b feature/my-feature`
3. **Make your changes**
4. **Test** your changes
5. **Commit** with clear message: `git commit -m "Add feature: description"`
6. **Push** to your fork: `git push origin feature/my-feature`
7. **Create a Pull Request** to the main repository

## Code Standards

- Follow Dart style guidelines
- Add comments for complex logic
- Update README if adding features
- Update CHANGELOG.md

## Pull Request Process

1. PRs require 1 approval before merging
2. All tests must pass
3. Code must be properly formatted
4. Description should explain the changes

## Questions?

Open an issue or contact the maintainers.
